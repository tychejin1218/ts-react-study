# 6~8주차: 실전 프로젝트

> 목표: 배운 것을 하나의 완성된 앱으로. **완성이 학습이다** — 배포까지 간다.
> 여기까지의 강의/문서 지식은 프로젝트를 완성해야 비로소 내 것이 된다.

## 프로젝트 선택 (하나만, 작게)

백엔드를 직접 만들 수 있다는 강점을 살린 주제 추천:

### 옵션 A: 관리자 대시보드 (추천)
직접 만든 Spring Boot API + React 어드민.
- 리소스 목록 (검색/필터/정렬/페이지네이션)
- 상세/수정 폼, 통계 카드 + 차트 1개 (recharts)
- 로그인 + 권한별 메뉴

### 옵션 B: 학습 노트 / 플래시카드 앱
- 카드 CRUD, 태그 필터, 복습 모드(간격 반복)
- localStorage → 백엔드 연동으로 확장

### 옵션 C: 사이드 프로젝트 아이디어가 이미 있다면 그것
단, 범위를 위 옵션 수준으로 잘라낼 것.

## 기술 스택 (고정 — 고민에 시간 쓰지 말 것)

```
Vite + React + TypeScript
├── 라우팅        react-router
├── 서버 상태     @tanstack/react-query
├── 클라이언트 상태  zustand (필요할 때만)
├── 폼           react-hook-form + zod
├── 스타일        tailwindcss (+ shadcn/ui 권장)
├── HTTP         axios 또는 fetch 래퍼
└── 배포          Vercel 또는 Cloudflare Pages
```

## 진행 체크리스트

### 1단계: 셋업 (반나절)
- [ ] Vite 프로젝트 생성, ESLint + Prettier 설정
- [ ] 폴더 구조 잡기 (기능 기준):
  ```
  src/
  ├── app/          # 라우터, 프로바이더, 전역 설정
  ├── features/     # 기능별 모듈 (posts/, auth/, ...)
  │   └── posts/
  │       ├── api.ts        # API 호출 + Zod 스키마
  │       ├── hooks.ts      # useQuery/useMutation 래핑
  │       └── components/
  ├── components/   # 공용 UI 컴포넌트
  ├── lib/          # API 클라이언트, 유틸
  └── types/        # 공용 타입
  ```
- [ ] API 클라이언트 + 에러 처리 공통 모듈
- [ ] 첫 배포 — **기능 없이도 먼저 배포부터** (CI/CD 감각, 이후 머지마다 자동 배포)

### 2단계: 핵심 기능 (1주)
- [ ] 목록 조회 (로딩 스켈레톤, 에러 화면, 빈 상태 포함)
- [ ] 상세 조회
- [ ] 생성/수정 폼 (RHF + Zod)
- [ ] 삭제 (확인 다이얼로그 + 캐시 무효화)

### 3단계: 완성도 (1주)
- [ ] 인증 (로그인/로그아웃/라우트 보호)
- [ ] 검색/필터 — 상태를 URL 쿼리 파라미터에 (새로고침해도 유지)
- [ ] 반응형 레이아웃 (모바일에서 깨지지 않는 수준)
- [ ] 404 페이지, 전역 에러 바운더리
- [ ] README 작성: 스크린샷, 실행 방법, 기술 선택 이유

### 4단계: 회고
- [ ] `notes/`에 회고 작성: 막혔던 것, 다시 한다면 다르게 할 것
- [ ] 코드 리뷰 요청 (동료, 커뮤니티, 또는 Claude에게)
