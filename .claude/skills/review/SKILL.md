# 학습 코드 리뷰 스킬

인자로 받은 경로의 코드를 리뷰한다. 예: `/review playground/ts`, `/review project/src/features/posts`. 인자가 없으면 git 상태를 확인해 최근 변경된 학습 코드를 대상으로 한다.

## 리뷰 관점

실무 코드 리뷰가 아니라 **교육용 리뷰**다. 목적은 코드를 고치는 게 아니라 학습자의 이해를 교정하는 것.

## 리뷰 구조 (반드시 이 순서로)

1. **잘한 점 먼저** — 구체적으로. 어떤 개념을 제대로 적용했는지 짚는다 ("판별 유니온으로 상태를 모델링한 것이 정확하다" 수준의 구체성)
2. **버그/오개념** — 동작이 잘못되거나 개념을 오해한 부분. 각 지적마다:
   - 무엇이 왜 문제인지 (Java 습관에서 온 것이면 그 점을 명시)
   - 관련 커리큘럼 항목 연결 (예: "3주차 '상태는 스냅샷' 참고")
3. **관용적(idiomatic) 개선 방향** — 동작은 하지만 React/TS 커뮤니티 관습과 다른 부분

## 핵심 규칙

- **코드를 직접 수정하지 않는다.** Edit/Write 금지. 수정 방향과 힌트만 제시하고 학습자가 직접 고치게 한다
- 각 지적은 "어느 파일 어느 부분"인지 `파일:줄번호`로 명확히
- 고칠 코드를 통째로 보여주지 말고, 패턴 수준의 짧은 예시(3~5줄)까지만
- 지적이 많으면 중요한 것 3개만 골라 우선순위를 매긴다 — 한 번에 다 고치게 하지 않는다
- 마무리: 리뷰에서 나온 오개념 중 가장 중요한 것 하나를 `notes/`에 정리하도록 권유하고, 고친 뒤 다시 `/review`를 요청하라고 안내한다
