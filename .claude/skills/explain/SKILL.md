# 개념 설명 스킬

인자로 받은 개념을 강사 방식으로 설명한다. 예: `/explain 구조적 타이핑`, `/explain useEffect 클린업`.

## 설명 구조 (반드시 이 순서로)

1. **백엔드 비유로 도입** — 학습자가 이미 아는 Java/Spring/JPA/HTTP 개념과 대응시켜 한 문단으로 감을 잡게 한다. 억지 비유는 하지 말고, 대응이 어긋나는 지점("단, 여기서부터는 다르다")도 명시한다
2. **핵심 원리** — 무엇이 문제였고 이 개념이 어떻게 해결하는지. 동작 방식을 정확하게
3. **실행 가능한 최소 예제** — 복붙해서 바로 돌려볼 수 있는 짧은 코드. TS Playground나 `playground/`에서 실행하는 방법 한 줄 안내
4. **흔한 실수** — 특히 Java 경험자가 빠지기 쉬운 함정 1~2개, 잘못된 코드 → 올바른 코드 대비로
5. **이해 확인 질문 1개** — 설명한 내용을 살짝 비튼 상황 제시. 턴을 끝내고 답을 기다린다

## 추가 규칙

- 설명 시작 전에 `curriculum/*.md`에서 이 개념이 어느 주차/체크박스에 해당하는지 찾아 짚어준다 (예: "이건 1주차 Day 3-4 범위다")
- 커리큘럼보다 한참 앞선 개념이면 설명은 하되, 지금 단계에서 알아야 할 깊이를 명시하고 나머지는 "N주차에서 다시"라고 범위를 잘라준다
- 학습자가 확인 질문에 답하면 채점·해설 후, 이 개념을 `notes/`에 TIL로 정리하도록 권유한다
