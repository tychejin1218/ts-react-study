# 퀴즈 스킬

학습자가 배운 내용을 퀴즈로 점검한다. 인자: `/quiz [주차|주제]` (예: `/quiz 1주차`, `/quiz 제네릭`). 인자가 없으면 `curriculum/*.md`의 체크박스에서 **최근에 완료된 항목들**을 찾아 그 범위에서 출제한다.

## 출제 규칙

- 총 **5문제**: 개념 문제 3개 + 코드 읽기 문제 2개
- 코드 읽기 문제는 "이 코드의 출력/타입 에러는?" 형식으로, 학습자가 흔히 틀리는 지점(함정)을 겨냥한다
- 난이도는 해당 커리큘럼 문서의 수준을 따른다. 아직 체크되지 않은(안 배운) 범위에서 내지 않는다
- 백엔드 개발자가 Java 습관 때문에 틀리기 쉬운 지점을 우선 출제한다

## 진행 방식

1. **한 번에 한 문제씩** 낸다. 문제를 내고 턴을 끝내 학습자의 답을 기다린다
2. 답변을 받으면: 정답 여부 → 왜 그런지 해설 → 관련 함정 한 줄. 그리고 다음 문제
3. 모르겠다고 하면 정답을 바로 주지 말고 힌트를 먼저 준다
4. 5문제가 끝나면 마무리 리포트:
   - 점수와 취약 지점 요약
   - 취약 지점별 복습할 커리큘럼 항목/링크
   - 틀린 문제는 `notes/`에 오답 노트로 정리하도록 권유

## 예시 문제 스타일

```ts
// Q. 다음 코드에서 컴파일 에러가 나는 줄과 이유는?
type Status = 'pending' | 'done';
let s: Status = 'pending';
s = 'in_progress';
```
